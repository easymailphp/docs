EasymailPHP

EasymailPHP is a PHP library for sending and receiving email using SMTP, IMAP, and the Gmail API.

Installation

Using Composer

Composer support is coming soon.

Manually

Copy the src directory onto your webserver and include the src/_load.php file, like this:

include 'src/_load.php';

Usage

SMTP

The SMTP class provides a simple implementation of the SMTP protocol for sending email. To use this class, you would create an instance of the class and pass in the hostname and port for your SMTP server, as well as the username and password for your email account. You would then call the sendEmail method on the class to send an email.

$smtp = new SMTP("smtp.example.com", 587, "username", "password");
$smtp->sendEmail("recipient@example.com", "Subject", "Body");

IMAP

The IMAP class provides a simple implementation of the IMAP protocol for receiving email. To use this class, you would create an instance of the class and pass in the hostname and port for your IMAP server, as well as the username and password for your email account.

$imap = new IMAPClass("mail.example.com", "username", "password");
$message = $imap->get_message(1);
$unread_messages = $imap->get_unread_messages();
$all_messages = $imap->list_all_messages();

Gmail API

The GmailAPI class provides a simple implementation of the Gmail API for sending email using OAuth authentication. To use this class, you would need to obtain a client_id, client_secret, and refresh_token for your Gmail account, as well as the token endpoint URL.

Obtaining a Client ID and Client Secret

To obtain a client_id and client_secret for your Gmail account, you'll need to create a new Google Cloud Platform project and enable the Gmail API for that project. Here are the steps to do this:

  1. Go to the Google Cloud Console and create a new project.
  2. Once you've created the project, select it from the project list, and then click on the "APIs & Services" menu option on the left-hand side of the page.
  3. Click on the "Enable APIs and Services" button at the top of the page.
  4. Search for "Gmail" in the search bar, and then select the "Gmail API" from the results.
  5. Click on the "Enable" button to enable the Gmail API for your project.
  6. Click on the "Credentials" menu option on the left-hand side of the page.
  7. Click on the "Create credentials" button at the top of the page, and then select "OAuth client ID" from the options.
  8. Follow the prompts to configure your OAuth client ID, selecting "Web application" as the application type.
  9. Once you've created your OAuth client ID, you'll see a page with your client_id and client_secret on it. Make note of these values, as you'll need them to authenticate with the Gmail API.

Obtaining a Refresh Token

In addition to the client_id and client_secret, you'll also need a refresh_token in order to authenticate with the Gmail API. Here's how to obtain a refresh_token:

  1. Go to the Google OAuth Playground and click on the "Settings" button in the top right corner.
  2. Check the "Use your own OAuth credentials" checkbox, and enter your client_id and client_secret in the appropriate fields.
  3. Close the settings dialog, and then select "Gmail API v1" from the list of APIs on the left-hand side of the page.
  4. Click on the "Authorize APIs" button to authorize the Gmail API for your account.
  5. Follow the prompts to authorize the API, and then click on the "Exchange authorization code for tokens" button to exchange the authorization code for an access token and refresh token.
  6. Make note of the refresh_token that is returned in the response. You'll need this value to authenticate with the Gmail API.

Using the GmailAPI Class

With your client_id, client_secret, refresh_token, and token endpoint URL in hand, you can now use the GmailAPI class to send email using the Gmail API. Here's an example:

$gmail = new GmailAPI("client_id", "client_secret", "refresh_token", "https://accounts.google.com/o/oauth2/token");
$gmail->sendEmail("recipient@example.com", "Subject", "Body");

iCloud

iCloud provides email services through the IMAP and SMTP protocols. To use iCloud email with EasymailPHP, you would use the iCloudIMAP and iCloudSMTP classes, which extend the IMAP and SMTP classes and set the hostname and port for iCloud's IMAP and SMTP servers.

$imap = new iCloudIMAP("imap.mail.me.com", 993, "username", "password");
$smtp = new iCloudSMTP("smtp.mail.me.com", 587, "username", "password");

Note that you would still need to provide valid iCloud email account credentials in order to use these classes to send and receive emails.
